- .freezeメソッド
    - そのオブジェクトを 変更不可にする。
    
    ```ruby
      ENDPOINTS = {
        areas: "/areas",
        breweries: "/breweries",
        brands: "/brands"
      }.freeze
    ```
    
    - 変更不可能としたオブジェクトを変更しようとするとエラーが出る
        
        ```ruby
        ENDPOINTS[:areas] = "/new_areas"
        # => FrozenError (can't modify frozen Hash)
        ```
        
- HTTP通信のタイムアウト時間の設定
    
    ```ruby
    OPEN_TIMEOUT = 10
    READ_TIMEOUT = 30
    
    http = Net::HTTP.new(url.host, url.port)
    http.use_ssl = true
    http.open_timeout = OPEN_TIMEOUT
    http.read_timeout = READ_TIMEOUT
    ```
    
    - `OPEN_TIMEOUT` :接続タイムアウト
        - サーバーに接続するまでの最大待機時間（秒）
        - 待っているもの
            - DNS解決
            - TCP接続確立
            - SSLハンドシェイク
    - `READ_TIMEOUT` :読み取りタイムアウト
        - 接続後、レスポンスが返ってくるまでの最大待機時間（秒）
        - 例)以下の場合、30秒待ってもレスポンスが無ければエラーとする
            - APIサーバーが重い
            - 処理が遅い
            - DBが詰まっている
- `find_or_initialize_by`と`find_or_create_by`の違い
    - `find_or_initialize_by`：検索して、なければ new（未保存）
    - `find_or_create_by`：検索して、なければ create（保存まで行う）

## **Tailwind CSS クラスソーターの導入（RustyWind）**

### 背景

Tailwind CSS のクラスの並び順がファイルごとにバラバラだったため、自動ソートの仕組みを導入したかった。

### ツール選定

以下のツールを比較検討した。

| ツール | Tailwind v4 対応 | `class: "..."` | ERB 安全性 | メンテナンス | 判定 |
| --- | --- | --- | --- | --- | --- |
| **RustyWind** | ○ | ○（custom-regex） | △（要対策） | 活発（51リリース） | **採用** |
| tailwind-sorter | ×（v3のみ） | ○ | ○ | 低（2リリース） | 見送り |
| Herb Formatter | ○ | ○ | ○（AST） | 活発 | 見送り（実験段階） |
| prettier-plugin-tailwindcss | ○ | × | ×（ERB非対応） | 活発 | 見送り |

**各ツールの見送り理由**

- **tailwind-sorter**: ソート順を YAML で手動定義する方式のため、Tailwind v4 で追加された約70〜80種のクラスに非対応。今後のバージョンアップのたびに手動メンテナンスが必要
- **Herb Formatter**: ERB 専用パーサー（AST ベース）で最も正確だが、実験的プレビュー段階。ERB 補間を含む class を破壊するバグ（[Issue #879](https://github.com/marcoroth/herb/issues/879)）あり
- **prettier-plugin-tailwindcss**: ERB ファイル（`.html.erb`）を公式サポートしておらず、`<% %>` 部分で余計な改行が入る問題あり

### RustyWind の課題と対策

**課題1: ERB 内の三項演算子が壊れる（Issue #124）**

```
<%# 入力 %>
<%= rate < sake_log.rating ? "bg-yellow-300" : "bg-stone-400" %>
<%# RustyWind 処理後（壊れる） %>
<%= rate < sake_log.rating ?"bg-yellow-300" : "bg-stone-400" %>
```

`?` 前のスペースが削除され、Ruby の構文エラーになる。

**対策**: 三項演算子をヘルパーメソッドに切り出し。

```ruby
# app/helpers/sake_logs_helper.rb
def star_rating_class(rate, rating)
  rate < rating ? "bg-yellow-300" : "bg-stone-400"
end
```

**課題2: Rust の正規表現エンジンが先読み（lookahead）非対応**

当初、ERB 補間を含む class 属性をスキップするために負の先読みを使おうとしたが、Rust の `regex` クレートでは先読み・後読みが一切使えなかった。

```
# エラーになる
class\\s*(?:=\\s*|:\\s*)"(?![^"]*(?:<%|#\\{))([^"]+)"
                      ^^^ look-around is not supported
```

**対策**: 先読みの代わりに、文字クラスで `<` と `{` を除外する方式に変更。

```
# 最終的な正規表現
class\\s*(?:=\\s*|:\\s*)"([^"<{]+)"
```

- `<` を除外 → ERB タグ（`<%= %>`、`<%# %>`、`<% if %>`）を含む class をスキップ
- `{` を除外 → Ruby 補間（`#{}`）を含む class をスキップ
- Tailwind の任意値（`bg-[#ff0000]`）は `<` も `{` も含まないため正常にマッチ

### 正規表現の設計

**対応パターンの検討**

今回採用した正規表現では以下の表のように正規表現がマッチし、ERB タグ ・Ruby 補間が含まれる場合のみ並び替え対象外となる。

| 対象 | マッチ | 仕組み |
| --- | --- | --- |
| `class="flex mt-4"` | ○ | 通常どおりマッチ |
| `class= "flex mt-4"` | ○ | `=\\s*` でスペースに対応 |
| `class ="flex mt-4"` | ○ | `class\\s*` でスペースに対応 |
| `class: "btn btn-primary"` | ○ | `:\\s*` でマッチ |
| `class="alert alert-<%= ... %>"` | × | `<` で停止 → 並び替えスキップ |
| `class: "mask #{color}"` | × | `{` で停止 → 並び替えスキップ |
| `class="bg-[#ff0000]"` | ○ | `#` は除外対象外なのでマッチ |
| `class="w-[50%]"` | ○ | `%` は除外対象外なのでマッチ |

### 導入内容

**1. RustyWind のインストール**

package.json の `devDependencies` に `rustywind` を追加し、`yarn install` を実行する。

```bash
# Docker 環境の場合
docker compose exec web yarn add --dev rustywind
```

これにより package.json に以下が追加される。

```json
{
  "devDependencies": {
    "rustywind": "^0.24.3"
  }
}
```

**2. npm スクリプトの追加**

package.json の `scripts` にソート実行用とチェック用のコマンドを追加する。

```json
{
  "scripts": {
    "tailwind:sort": "rustywind --write --custom-regex 'class\\\\s*(?:=\\\\s*|:\\\\s*)\\"([^\\"<{]+)\\"' app/views/",
    "tailwind:check": "rustywind --check-formatted --custom-regex 'class\\\\s*(?:=\\\\s*|:\\\\s*)\\"([^\\"<{]+)\\"' app/views/"
  }
}
```

| スクリプト | 用途 |
| --- | --- |
| `tailwind:sort` | クラスを並び替えてファイルを上書き保存 |
| `tailwind:check` | ソート順のチェックのみ（未ソートがあれば exit code 1） |

**3. CI（GitHub Actions）の設定**

`tailwind_sort_check` ジョブを追加し、PR 作成時に自動でソート順をチェック。

**使い方**

```bash
# ソート実行
```
