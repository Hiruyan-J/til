# Rails基礎でdbを確認・SQLで操作する方法
Rails基礎で、デバッグの為DBを確認したかったので、DBを確認・操作する方法を調べました。
  
**【調べた背景】**
カリキュラムでコードを変更したところ、エラー画面は出ないが思った通りの挙動をしなかったためです。
- テーブルにレコードが追加されているのかを確認したい
- 追加したレコードを削除したい
ということで、MySQLでDBを確認する方法を調べました。
  
私と同じく、Rails基礎(もしかしたらRails入門でも使える？)でテーブルの中身の確認・操作を行いたいというニーズがあるかと思い、調べた内容を以下にまとめましたのでシェアします。
___
**【dbを確認・SQLで操作する方法】**
1. 事前準備
	- MySQLのパスワードの入力を求められるので、パスワードを確認する
		- config/database.ymlに記載あり
			```
			default: &default
			  adapter: mysql2
			  encoding: utf8mb4
			  pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
			  username: root
			  password: password #←ここの部分
			  host: db
		  ```  
1. MySQLに接続(ここからは、データベース・SQL基礎Ch2と同じ操作です)
  ```
  $ docker compose exec db bash
	# mysql -u root -p
	Enter password: #←ここで先ほどのパスワードを入力
  ```  
1. DB一覧確認：`mysql> show databases;`
	```
	+----------------------------------+
	| Database                         |
	+----------------------------------+
	| information_schema               |
	| mysql                            |
	| performance_schema               |
	| sys                              |
	| v3_basic_rails_basic_development |
	| v3_basic_rails_basic_test        |
	+----------------------------------+
	```  
1. DB選択：`mysql> use ○○;` …DB一覧の中から選ぶ
  - Rails基礎なら `v3_basic_rails_basic_development` 
1. テーブル一覧を確認：`mysql> show tables;`
	```
	+--------------------------------------------+
	| Tables_in_v3_basic_rails_basic_development |
	+--------------------------------------------+
	| ar_internal_metadata                       |
	| schema_migrations                          |
	| users                                      |
	+--------------------------------------------+
	```  
1. 以降、SQLで操作可能
	1. 例)テーブルの中身を確認：`mysql> select * from users;`
		- 出力結果は添付画像。画像の通り、テーブルの見た目で出力できます。
1. 終了時は、 `exit`で抜けます。
