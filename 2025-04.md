**【課題を通して学んだこと】**（知識・タスク分解などの振り返り）  
- .html.slimファイルについて
	- if文はネストで表現する必要があり、最後の`end`を書いてしまうとエラーが発生する
- local_assigns
	- `locals:`オプションに渡した個別のキーは、`local_assigns`ヘルパーメソッドを使うことでパーシャルのローカル変数としてアクセスできるようになる
	- local_assignsを使用することによって、変数が無い場合、エラーを返さずnilを返せる
- render_to_string
	- render呼び出しの正確な結果をブラウザを使わずに調べたい場合は、render_to_stringを利用できる
	- このメソッドの振る舞いは、レンダリング結果をブラウザに返さずに文字列を返す点を除けば、renderと完全に同じである
- render layout: false
	- デフォルトでは、renderでViewファイルを読み込むと、共通のビューファイル(レイアウトファイル)も読み込まれてしまう。(application.html.erbなど)
	- `layout: false` で、レイアウトファイルを読み込ませないようにできる。
	- [Railsガイド](https://railsguides.jp/v7.1/layouts_and_rendering.html#layout%E3%82%AA%E3%83%97%E3%82%B7%E3%83%A7%E3%83%B3)
- Active Strage
	- Active Storageは、Amazon S3、Google Cloud Storage、Microsoft Azure Storageなどのクラウドストレージサービスへのファイルのアップロードや、ファイルをActive Recordオブジェクトにアタッチする機能を提供している
	- Active Storageは、アプリケーションのデータベースで active_storage_blobs、active_storage_variant_records、active_storage_attachmentsという名前の3つのテーブルを使う
	- [Railsガイド](https://railsguides.jp/v7.0/active_storage_overview.html#active-storage%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6)
- ポリモーフィック関連付け
	- polymorphic: true
	- ポリモーフィック関連付けを使うと、ある1つのモデルが他の複数のモデルに属していることを、1つの関連付けだけで表現できる
	- [Railsガイド](https://railsguides.jp/v7.0/association_basics.html#%E3%83%9D%E3%83%AA%E3%83%A2%E3%83%BC%E3%83%95%E3%82%A3%E3%83%83%E3%82%AF%E9%96%A2%E9%80%A3%E4%BB%98%E3%81%91)
- 単一テーブル継承(STI)
	- 複数モデルが持っている情報が同じ場合、1つのテーブルで複数モデルのテーブルを表現することが出来る
	- 今回は、Taxonomiesテーブルだけで以下のテーブルを作成している
		- Author
		- Category
		- Tag
