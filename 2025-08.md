**【課題を通して学んだこと】**（知識・タスク分解などの振り返り）  
- Javaの特徴
    - Javaの主な特徴は2つ
        - オブジェクト指向
        - プラットフォーム非依存
            - JVMによってOSの違いを吸収してくれる
            Java仮想マシンのこと
            - JDK
                - Javaの開発環境
                - JDKとして含まれているもの
                    - コンパイラ、JVM、その他ツール
- Javaの開発の流れ
    - ソースプログラムの作成
        - `○○.java`
    - コンパイル
        - コマンドプロンプトで `javac ○○.java` でコンパイル
        - `○○.class` が生成される
    - 実行
        - `java クラス名` (拡張子は入力しない)
- プログラムの構成
    - Javaのプログラムは、**クラス**の集まり
    - クラス名とファイル名は合わせること
    - クラスの内容は`{ }` 間(ブロック)に書く
    - 処理のまとまりは**メソッド**
    - **mainメソッドから実行**される
    - 処理文は`;`で終わらせる
    - 書き方
        
        ```java
        class クラス名{
        	public static void main(String args[]){
        		処理文
        	}
        }
        ```
        
- 変数
    - 変数宣言
        
        ```java
        int num;
        ```
        
    - 複数同時に宣言
        
        ```java
        int month, day;
        ```
        
    - 変数宣言時に初期化
        
        ```java
        int age = 20;
        ```
        
- データ型
    - データ型は3,000種類
        - 基本データ型は8種類
            - intやlong、flootやchar、booleanなど
        - それ以外は、参照型
            - Stringなど
            - 変数には、データ自体のアドレス値(参照値)だけを格納している。
- コマンドラインからデータを受け取る方法
    - javaコマンド実行時に、クラス名の後ろで引数を渡す
    
    ```java
    java クラス名 Hello Java
    ```
    
- switch文
    - 注意点
        - `braak;` が無い場合、呼ばれた処理文以下の処理文が実行される(`default:`以外)



# ** 8/23(木) **
### 【学習時間】
1h(累計:905h/1,000h)
  
### 【学習内容】
- Java

### 【課題を通して学んだこと】
- while文・for文
- オブジェクト指向
    - クラスは2種類
        - 実行用クラス
        - 設計図クラス
            - 属性と操作を持つ
- 設計図クラス
    - 変数の定義は、クラス名のすぐ下で行うこと
        - クラスで使う変数を**メンバ変数**という
        - メンバ変数は、そのクラス内でのみ使用可
    - メソッドの記載方法
        
        ```java
        戻り値の型 メソッド名(引数リスト){
        	(処理)
        }
        ```
        
    - 呼び出す際に引数は無くても`メソッド名()`のように`()` を記載する必要がある。
    - 戻り値の注意点
        - メソッド名に書かれている戻り値の型と`return`で返す値の型を合わせる必要がある。
    - メソッド内で宣言した変数は、メソッド内でのみ使用可
- オブジェクトの使い方
    1. 生成(インスタンス化)
        - `クラス名 オブジェクト名 = new クラス名();`
        - 例)`Student stu1 = new Student();`
    2. 変数・メソッドを利用
        - `オブジェクト名.変数名`
        - `オブジェクト名.メソッド名(引数)`


# ** 8/24(日) **
### 【学習時間】
4.6h(累計:911h/1,000h)
  
### 【学習内容】
- Java
  
### 【課題を通して学んだこと】
- オーバーロード
    - クラス内に味名前で引数の型や数が違うメソッドを複数定義すること
    - オーバーロードのメリット
        - 作る側
            - プログラムが分かりやすくなる
            - メソッドを沢山作っても、名前が被ってもOKなので、名前を考えるのが楽になる
        - 呼び出す側
            - 例えば`println`メソッド
                - オーバーロードが無いと：渡す引数がint型、string型…でそれぞれメソッド名が変わってしまう。
                - オーバーロードがあると：渡す引数が何であろうと、`println`メソッドでOK
- コンストラクタ
    - オブジェクトの初期化のために使われる特殊なメソッド
        - ルール
            - 名前が**クラス名**と同じ
            - **戻り値**を持たない
            - `new クラス名(引数)`で呼び出す
    - コンストラクタを作成した場合、デフォルトコンストラクタは生成されない
- static
    - インスタンスの中の変数にcounterが生成されるので、毎回1になってしまう。
    - インスタンス共通で使用できる変数・メソッド…staticキーワードで作る
    - static変数・メソッドは、オブジェクトを生成していなくても利用できる。
    - `利用方法→クラス名.変数(or メソッド)名`
- カプセル化
    - `アクセス修飾子`でクラス・メンバ変数・メソッドの公開範囲を指定できる
    - カプセル化のメリット
        - **代入前処理**ができる
            - 代入される値が正しい値になっているかを確認できる
                - 例)0～100点の範囲内か？
        - **修正範囲**を狭めることが出来る
            - privateの変数は、他のクラスでは現れないので、同じクラスのプログラムだけ修正すればよくなる

# ** 8/27(水) **
### 【学習時間】
1.2h(累計:913h/1,000h)
  
### 【学習内容】
- Java
- 基本情報

### 【課題を通して学んだこと】
- 継承
    - 既存のクラスをもとに変数やメソッドを追加したクラスを作ること
        - 親：スーパークラス
        - 子：サブクラス
    - コンストラクタは継承しない
- オーバーライド
    - サブクラスでスーパーのメソッドを再定義すること
    - 似ている名前：オーバーロード
        - 同じクラスの中に同じ名前で複数メソッドを定義できる
- thisとsuper
    - thisとsuperによるメンバ変数・メソッドの呼び出し
        - this
            - 自オブジェクトのメンバ変数・メソッドの呼び出し
            
            ```java
            String name;
            void setName(String name){
            	this.name = name;
            }
            ```
            
            - `this.name` でメンバ変数の`name` を指定
                - 引数の`name` とメンバ変数の`name` を区別できる。→引数をメンバ変数と同じ`name` にできる
        - super
            - スーパークラスのメンバ変数・メソッドの呼び出し
        - 引数名を考えなくて良い→コードを自動で書いてくれる
    - thisとsuperによるコンストラクタの呼び出し
        - `this(引数)`
        - `super(引数)`
