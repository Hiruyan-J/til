タスク管理ページでviewファイルを作成中です。
まだ、画面でDBへのレコードの追加ができないので、カリキュラムにはないですがconsoleでレコードを追加しながら表示を確認しながら進めています⛺
そんな中、気になったこと
- consoleでのモデルのカラム名の確認法について
	- モデル名.column_names
	- モデル名.attribute_names
 の2つあるが、違いは何かが気になった。  

ChatGPT先生曰く、
- Task.column_names: データベースに存在するカラム名を知りたいときに使う。
- Task.attribute_names: モデルの属性全体（仮想属性も含む）を知りたいときに使う。
- 仮想属性を追加していない場合は、両者の結果は基本的に同じです。ただし、アプリケーションが複雑になるにつれて、attribute_names のほうが広い範囲をカバーします。  

| **特徴** | `column_names` | `attribute_names` |
| ---- | ---- | ---- |
| **対象** | データベースのカラム | モデルが扱えるすべての属性 |
| **仮想属性を含むか** | 含まない | 含む |
| **利用シーン** | データベースの物理的なカラムを確認 | アプリケーションで扱える属性全体を確認 |

(まとめ抜粋)
とのことでした。
