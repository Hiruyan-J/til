### 【エラー対応記録】
** 発生状況 **
git clone後初めてブラウザでアプリにアクセスしたところエラーが発生

** エラーの内容 ** 
`Sprockets::Rails::Helper::AssetNotFound in Posts#index`
Showing /myapp/app/views/layouts/application.html.erb where line #18 raised:
The asset "application.js" is not present in the asset pipeline.
```ruby
    <%= stylesheet_link_tag "tailwind", "inter-font", "data-turbo-track": "reload" %>
    <%= stylesheet_link_tag "application", "data-turbo-track": "reload" %>
    <%= javascript_include_tag "application", "data-turbo-track": "reload", type: "module" %>
  </head>

  <body>
```
（↑のコードの内、3行目でエラー発生）

** エラー概要 ** 
`application.js` が Sprockets のアセットパイプラインに見つからない

** 原因 ** 
今回は、git clone後、初めてWebアプリをブラウザで開いた。
git clone後、`docker compose exec web bin/dev` を行っていなかったため、`esbuild` を一度も動作させておらず、JavaScript アセットが `app/assets/builds` に出力されていない状態だったため、エラー発生

** 対処 ** 
- `docker compose exec web bin/dev` を実施
    - `bin/dev` を実行して `esbuild` を動かしたことで、
        - JavaScript アセットが `app/assets/builds` に出力され
        - Rails がそれを Sprockets 経由で認識できるようになる
        - →エラー解消

** 参考 ** 
- esbuildとは    
    > esbuild は Webpack に代表されるモジュールバンドラの 1 つで、複数の Javascript ファイルなどを 1 つのファイルにまとめるなどの機能がある
    >     
    https://qiita.com/Tsukina_7mochi/items/0aa38da6b9b4dd22eacd
